# ── LITE install profile — for the real 4 GB deploy PC (Dell Inspiron 3268) ──
#
# This machine (4 GB RAM, i3-7100, no GPU) runs OnnxTR OCR (docTR models on ONNX
# Runtime, NO PyTorch) — core/hardware.py auto-selects it below
# OCR_RAM_THRESHOLD_GB. PyTorch + docTR are NEVER loaded there. Installing them
# would waste ~2 GB of the ~52 GB free disk for zero benefit, so this profile
# deliberately OMITS `python-doctr` and does NOT install torch/torchvision.
#
#   pip install -r requirements-lite.txt
#
# The scanned-document path works via OnnxTR (grid detection + deep word
# reading, no Tesseract). CSV/Excel/typed-PDF need no OCR at all. If this box is
# ever upgraded to >= 6 GB RAM and you want docTR accuracy, install the full
# requirements.txt instead (adds python-doctr + CPU torch) — no code change.
#
# Tesseract is NOT a dependency anymore. The code still uses pytesseract for a
# couple of cheap helpers IF the binding happens to be installed, but every
# call site degrades to the OnnxTR deep engine when it is absent — so nothing
# here installs it.

# Runtime for the custom on-device digit OCR model (core/ocr_onnx.py). CPU-only,
# ~15-50 MB, numpy-2 compatible — NOT PyTorch. Lets the trained crnn.onnx read
# account/mobile digits accurately on the 4 GB CSP box. This same runtime also
# powers OnnxTR below.
onnxruntime==1.19.2
# OnnxTR = docTR's detection+recognition models on ONNX Runtime (NO PyTorch).
# This is the ACCURATE scanned-OCR engine for the 4 GB CPU box: detection finds
# every table row (Tesseract could not) and reads names; account/mobile digits
# still come from the custom crnn.onnx above. ~700 MB peak, fits 4 GB. Models are
# BUNDLED in core/models/ (db_mobilenet_v3_large + crnn_mobilenet_v3_small), so
# nothing is downloaded at run time (DPDP-safe). Base package (not [cpu]) so it
# reuses the onnxruntime pinned above and does NOT pull a second opencv. Its
# small deps (scipy, pyclipper, rapidfuzz, ...) are resolved automatically;
# pypdfium2 4.30.0 above already satisfies it. This is the OCR engine — there is
# no Tesseract fallback anymore.
onnxtr==0.8.1
