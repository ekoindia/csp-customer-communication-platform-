# ── CSP install profile — server-OCR client (real 4 GB Dell Inspiron 3268) ──
#
# OCR does NOT run on the CSP machine. Scanned PDFs/images are encrypted and
# sent to the Eko central OCR server (SERVER_OCR_ENABLED=1 in .env → see
# config.py ADMIN_API_BASE); the server does the deep-learning OCR and returns
# an Excel that is imported straight into the review screen. So the CSP box
# needs NO OCR model files and NO OCR runtime:
#
#   • NO onnxtr / onnxruntime  (deep OCR runs on the server, not here)
#   • NO torch / python-doctr  (never on a 4 GB box)
#   • NO Tesseract             (dropped as a dependency entirely)
#   • NO bundled core/models/*.onnx shipped to the CSP (the installer excludes
#     them — they exist only for the server / an optional on-prem OCR box)
#
# This is what makes the install small and fast: the CSP only downloads a
# handful of light pure-Python/wheels packages — no ~700 MB model runtime, no
# scipy build, no Tesseract winget download.
#
#   pip install -r requirements-lite.txt
#
# CSV / Excel / typed-PDF uploads are parsed locally (100% accurate, no OCR at
# all). Only SCANNED PDFs/images use the server. If you ever want a fully
# offline on-prem OCR box instead of the server, install requirements.txt
# (adds python-doctr + torch) or add onnxtr+onnxruntime back — no code change,
# core/hardware.py auto-detects what is present.

# (pytest is NOT a runtime dependency — it lives in requirements-dev.txt so the

#  requirements-dev.txt.)
# Loads local secrets/config from a .env file (never committed — see
