New CSP setup

The app installs straight from the public GitHub repo — you do NOT need to upload anything here. To onboard a CSP, just issue their key and download their ready-to-send setup file from the API Keys page.

Onboard a new CSP — 3 steps

1
Go to API Keys → enter the CSP's code → Issue.
2
On that CSP's row, click Download CSP_Setup.bat — it already has the GitHub link + CSP ID + API key baked in.
3
Send that ONE file to the CSP. They double-click it → everything installs into C:\CSP_Platform → they set their own login + branch on first run.

No zip to build, no package to upload, no key to send separately.

Optional: self-host the install package

Only if you want the app served from THIS server instead of GitHub (e.g. an air-gapped setup). Normally leave this empty — the setup file already points at GitHub.
